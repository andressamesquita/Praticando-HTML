HTML Semântico

O HTML semântico tem como objetivo descrever o significado do conteúdo presente em documentos HTML, tornando-o mais claro tanto para programadores quanto para browsers e outras engines que processam essa informação.

Antigamente, era necessário usar div para criar cabeçalhos, rodapés ou seções, por exemplo. Hoje, com o HTML5 existem tags específicas para esses elementos, onde as principais são apresentadas a seguir:

Header

O header é utilizado para representar o cabeçalho de um documento ou seção declarado no HTML. Nele podemos inserir elementos de h1 a h6, até elementos para representar imagens, parágrafos ou mesmo listas de navegação.

Exemplo do uso:

<header id="header" class="header">
<div id="detalhes-site">
</div id="logo">
<img src="/imagens/html5-logo.png" alt="html 5 logo">
</div>
<div id="info">
<h1>HTML - Header</h1>
<h2>Treinando HTML</h2>
</div>
</div>
<nav id="menu">
<ul>
<li><a href="/index.html">Home</a></li>
<li><a href="https://www.google.com">Site do Google</a></li>
<li><a href="https://www.youtube.com">Site do YouTube</a></li>
</ul>
</nav>
</header>

Section

O elemento section representa uma seção dentro de um documento e geralmente contém um título, o qual é definido por meio de um dos elementos entre h1 e h6. Podemos utilizar o section, por exemplo, para descrever as seções/tópicos de um documento.

Exemplo do uso:

<section>
<h2>Section</h2>
<p>O elemento section representa uma seção dentro de um documento e geralmente contém um título, o qual é definido por meio de um dos elementos entre h1 e h6.</p>
</section>

Article

Utilizamos o elemento atricle quando precisamos declarar um conteúdo que não precisa de outro para fazer sentido em um documento HTML, por exemplo, um artigo em um blog. É recomendado identificar cada atricle com um título.

Exemplo do uso:

<article>
<h3>Titulo do artigo 1</h3>
<p>...</p>
</article>
<article>
<h3>Titulo do artigo 2</h3>
<p>...</p>
</article>

Nav

O elemento nav é utilizado quando precisamos representar um agrupamento de links de navegação, que, por sua vez, são criados com os elementos ul, li e a.

Exemplo do uso:

<nav>
<ul>
<li><a href="http://#">pagina 1</a></li>
<li><a href="http://#">pagina 2</a></li>
<li><a href="http://#">pagina 3</a></li>
</ul>
</nav>

Aside

O elemento aside é utilizado quando precisamos criar um conteúdo de apoio/adicional ao conteúdo principal. Por exemplo, ao falar de HTML semântico, podemos indicar ao leitor outros conteúdos sobre a linguagem HTML como sugestão de leitura complementar.

Exemplo do uso:

<aside>
<nav>
<ul>
<li><a href="#">link 1</a></li>
<li><a href="#">link 2</a></li>
<li><a href="#">link 3</a></li>
</ul>
</nav>
<aside>

Main

O elemento main especifica o conteúdo principal e, consequentemente, de maior relevância dentro da página. Para ser considerada bem construída, uma página deve apresentar apenas um conteúdo principal.

Exemplo do uso:

<main>
<h2>Titulo</h2>
<p>...</p>
<article>
<h3>Titulo do artigo</h3>
<p>...</p>
</article>
</main>

Figure

O elemento figure é uma marcação de uso específico para a inserção de uma figura. Para incluir a descrição dessa figura, podemos utilizar o elemento figcaption.

Exemplo do uso:

<figure>
<img src="/imagens/html5-logo.png" alt="html 5 logo">
<figcaption>Logo do HTML5</figcaption>
</figure>

Footer

O elemento footer representa um rodapé de um documento, como a área presente no final de uma página web. Normalmente é utilizado para descrever informações de autoria, como nome e contato do autor, e data de criação do conteúdo.

Exemplo do uso:

<footer>
<p>ADS - Programação para Internet</p>
<p>Teresina - PI, outubro de 2018</p>
</footer>

Visualmente, o uso dessas tags não trará diferença alguma, mas um HTML semântico carrega significado independente da sua apresentação visual. Isso é particularmente importante quando o conteúdo for consumido por clientes não visuais. Há vários desses cenários:

  • Um usuário cego poderia usar um leitor de tela para ouvir sua página. Neste caso, a estrutura semântica do HTML é essencial para ele entender as partes do conteúdo;
  • Robôs de busca como o Google também são leitores não visuais da sua página. Sem um HTML semântico, o Google não consegue, por exemplo, saber que aquilo é um menu e que deve seguir seus links. Ou que determinada parte é só um rodapé informativo, mas não faz parte do conteúdo principal.
Então é muito importante o uso desses recursos presentes no HTML5
